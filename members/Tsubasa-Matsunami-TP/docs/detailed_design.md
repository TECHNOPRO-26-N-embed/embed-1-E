> ※ 基本設計書（2-2 関数一覧）で定義した各関数の「中身」を設計します。

### `setup()` — 初期化処理

```
【処理の流れ】

1. ピンモードを設定する
   - PIN_BUTTON_GU      → INPUT_PULLUP
   - PIN_BUTTON_CHOKI   → INPUT_PULLUP
   - PIN_BUTTON_PA      → INPUT_PULLUP
   - PIN_LED            → OUTPUT
   - PIN_BUZZER         → OUTPUT

2. 変数の初期化を行う
   - currentState を「入力待機」に設定
   - playerHand を -1 に設定
   - cpuHand を -1 に設定
   - gameResult を -1 に設定
   - winCount を 0 に設定
   - comboCount を 0 に設定

3. Serial.begin(9600) を実行する
   - デバッグ用

4. 起動確認を行う
   - LEDを約1秒点灯する
   - ブザーを短く鳴らす
   - LEDとブザーを停止する

5. ランダム関数用の初期化を行う
   - randomSeed() を使用して乱数初期化を行う
```
### `loop()` — メインループ

> ※ loop() は「状態ごとに何をするか」だけ書く。細かい処理は各関数に任せる。

```
【処理の流れ】

＜毎ループ実行すること＞
  - 現在時刻を取得する: now = millis()
  - ボタン入力状態を確認する
  - 必要に応じてLEDやブザー出力を更新する

＜currentState が 0（入力待機）のとき＞
  - inputHand() を呼び出してボタン入力を確認する
  - グー・チョキ・パーのいずれかが入力されたら
    → playerHand に値を保存する
    → currentState = 1（勝敗判定）

＜currentState が 1（勝敗判定）のとき＞
  - cpuSelectHand() を呼び出してCPUの手を決定する
  - judgeResult() を呼び出して勝敗判定を行う
  - 判定結果を gameResult に保存する
  - resultStartTime に現在時刻を保存する
  - currentState = 2（結果表示）

＜currentState が 2（結果表示）のとき＞
  - displayResult() を呼び出してLEDやブザーで結果表示を行う
  - 約2秒経過したら
    → playerHand を初期値へ戻す
    → cpuHand を初期値へ戻す
    → gameResult を初期値へ戻す
    → currentState = 0（入力待機）

＜currentState が 3（エラー）のとき＞
  - エラー用LED表示やブザー出力を行う
  - 一定時間後に currentState = 0（入力待機）へ戻す
```

### `関数名()` — （役割を1行で書く）

**inputHand()：** プレイヤーの手を入力する

**引数：**  なし

**戻り値：** int 型

```
【処理の流れ】
1. グーボタンの状態を確認する
2. チョキボタンの状態を確認する
3. パーボタンの状態を確認する
4. 押されたボタンに対応する値を返す
   - 0:グー
   - 1:チョキ
   - 2:パー
5. 入力がない場合は -1 を返す

【エラー・異常ケース】
- 複数ボタンが同時に押された場合:
  入力を無効として -1 を返す

- 前回入力から50ms未満の場合:
  チャタリングとして無視する
```

**cpuSelectHand()：** CPUの手をランダムに決定する

**引数：**  なし

**戻り値：** int 型

```
【処理の流れ】
1. random() を使用して 0～2 の乱数を生成する
2. 生成した値をCPUの手として返す
   - 0:グー
   - 1:チョキ
   - 2:パー

【エラー・異常ケース】
- 想定外の値が生成された場合:
  -1 を返してエラー状態へ移行する
```

**judgeResult()：** 勝敗を判定する

**引数：**  
playerHand (int) : プレイヤーの手

cpuHand (int) : CPUの手

**戻り値：** int 型

```
【処理の流れ】
1. playerHand と cpuHand を比較する
2. あいこの場合は 0 を返す
3. プレイヤー勝利の場合は 1 を返す
4. プレイヤー敗北の場合は 2 を返す

【エラー・異常ケース】
- playerHand または cpuHand が範囲外の場合:
  -1 を返してエラー状態へ移行する
```

**displayResult()：** 勝敗結果を表示する

**引数：**  
gameResult (int) : 勝敗結果

**戻り値：** void 型

```
【処理の流れ】
1. gameResult の値を確認する
2. 勝利時:
   - LEDを点灯する
   - 明るい効果音を鳴らす
3. 敗北時:
   - LEDを点滅させる
   - 低い効果音を鳴らす
4. あいこの場合:
   - LEDを短く点滅する
   - 短い通知音を鳴らす

【エラー・異常ケース】
- gameResult が範囲外の場合:
  エラー表示を行う
```

**updateOutput()：** LEDとブザー出力を更新する

**引数：**  
state (int) : 勝敗結果

**戻り値：** void 型

```
【処理の流れ】
1. currentState を確認する
2. 入力待機状態:
   - LEDを点灯状態にする
3. 結果表示状態:
   - 勝敗結果に応じた出力を行う
4. エラー状態:
   - LED点滅と警告音を出す

【エラー・異常ケース】
- state が未定義の場合:
  エラー状態へ移行する
```

## 3. 重要ロジックの詳細設計

### 3-1. チャタリング防止（デバウンス処理）

> ※ ボタンを使う場合は必ず設計してください。

```
【考え方】
  ボタンが押されたとき、50ms以内の連続入力は
  同じ1回の押下として扱い、誤入力を防止する。

【処理の流れ】
  1. グー・チョキ・パーの各ボタン状態を digitalRead() で取得する
  2. 現在時刻 millis() を取得する
  3. 前回入力時刻 lastButtonTime との差を計算する
  4. 経過時間 < DEBOUNCE_DELAY（50ms）の場合:
     → チャタリングと判断して入力を無効化する
  5. 経過時間 ≥ DEBOUNCE_DELAY の場合:
     → 正常な入力として playerHand を更新する
  6. lastButtonTime を現在時刻へ更新する

【必要な変数（Section 1 に追加済みか確認）】
  lastButtonTime : unsigned long
  // 前回の入力時刻

  DEBOUNCE_DELAY : const int = 50
  // チャタリング判定時間（ms）

【補足】
  INPUT_PULLUP を使用するため、
  ボタンが押されていない状態は HIGH、
  押された状態は LOW として判定する。
```

---

### 3-2. millis() を使ったタイマー管理

```
【考え方】
  「前回処理を実行した時刻」を保存しておき、
  「現在時刻 − 前回時刻 ≥ 指定時間」となった場合のみ処理を実行する。

  delay() を使用すると、その間すべての処理が停止するため、
  ボタン入力監視や状態管理を止めないために millis() を使用する。

【処理の流れ（結果表示時間管理）】
  1. now = millis() を取得する
  2. now - resultStartTime >= RESULT_DISPLAY_TIME か確認する
  3. 条件を満たした場合:
     - 結果表示を終了する
     - playerHand を初期値へ戻す
     - cpuHand を初期値へ戻す
     - gameResult を初期値へ戻す
     - currentState = 0（入力待機）へ戻す
  4. 条件を満たさない場合:
     - 結果表示を継続する

【処理の流れ（チャタリング対策）】
  1. now = millis() を取得する
  2. now - lastButtonTime >= DEBOUNCE_DELAY か確認する
  3. 条件を満たした場合:
     - ボタン入力を有効とする
     - lastButtonTime を更新する
  4. 条件を満たさない場合:
     - チャタリングとして入力を無効化する

【自分のシステムで millis() を使う処理】
  ・ボタン入力のチャタリング防止（50ms）
  ・勝敗結果の表示時間管理（約2秒）
  ・LEDやブザーの状態更新
  ・入力待機中の状態監視
```
